# 화면 검증: 통합 인터랙티브 미니홈피 (serve.py)

> 기존 메인화면(클래식 레이아웃)의 각 메뉴/탭을 실제 동작으로 활성화.

## 레이아웃

- 공통 셸: 상단 TODAY/TOTAL · 좌측 프로필(미니미·도토리·일촌·사진·쪽지) · 우측 세로탭 8개.
- 중앙 패널이 탭(`/?tab=...`)에 따라 교체된다.

## 활성화된 탭 ↔ 도메인

| 탭 | 화면 | 인터랙션(POST) |
| --- | --- | --- |
| home | 미니룸 + 최근 방명록 | (읽기) |
| profile | 미니미·도토리·일촌·각 카운트 | `/acorn/charge` 도토리 충전 |
| diary | 다이어리 목록(공개범위 뱃지) | `/diary/write`·`/diary/delete` |
| photo | 사진첩 그리드(대표 표시) | `/photo/upload`·`/photo/cover`·`/photo/delete` |
| board | 게시판 목록(최신순) | `/board/write`·`/board/delete` |
| guestbook | 방명록(비밀글) | `/guestbook/write`·`/guestbook/delete` |
| message | 받은 쪽지함(안읽음) | `/message/send`·`/message/read` |
| ilchon | 일촌 목록 | `/ilchon/add`·`/ilchon/break` |

## 런타임 검증 (HTTP)

- 8개 탭 모두 `GET /?tab=X` → 200 + 올바른 h2 렌더.
- 다이어리 작성 → 글 수 2→3.
- 방명록 비밀글 작성 → secret 뱃지 노출.
- 도토리 충전 500 → 잔액 1200→1700.
- 일촌 추가 → 3→4.
- 쪽지 발송 → 안읽음/전체 3/3.
- 전부 303 redirect 후 해당 탭에 즉시 반영, 한글 UTF-8 정상.

## 단위 게이트

- `python proof/run_proof.py` → exit 0, 46/46 PASS (도메인 로직 불변).

## 메인화면(클래식 레이아웃)

- 참조 이미지와 동일한 싸이월드 클래식 룩: 상단 TODAY/TOTAL · "사이좋은 사람들" · URL,
  좌측 미니미 아바타 + 대표사진 박스 + 도토리 + 일촌 + CYWORLD②, 중앙 Updated news 메뉴그리드
  + Mini Room + What friends say(방명록), 우측 세로탭.
- 중앙 메뉴그리드/좌측 일촌/우측 탭이 모두 링크 → 해당 기능 탭으로 이동.

## 시점 전환 (주인/일촌/방문자)

- 상단 토글(`/switch?who=...`)로 현재 보는 시점을 바꾼다. `home.main(OWNER, viewer)` 및 각
  도메인 `list_for(OWNER, viewer)`가 공개범위를 그대로 적용 → 화면에 반영.

| 시점 | 사진첩 | 다이어리 | 도토리 | 쪽지함 | 다이어리 탭 |
| --- | --- | --- | --- | --- | --- |
| 주인(과니) | 3 | 3 🔒 | 1200 | 2/2 | 전체 |
| 일촌(병익) | 2 | 2 🔒 | 비공개 | 비공개 | 일촌공개까지 |
| 방문자 | 1 | 0 | 비공개 | 비공개 | 차단("일촌만 볼 수 있어요") |

- 주인 전용 액션(사진 업로드·다이어리 작성·도토리 충전·일촌 맺기·쪽지함 열람)은 방문자 시점에서
  폼이 숨겨지고 서버에서도 차단(POST 가드). 방명록·게시판·쪽지 보내기는 방문자도 가능.
- HTTP 검증: 시점별 사진 3/2/1, 다이어리 3/2/0(섹션 비공개), 도토리/쪽지 주인만 — 전부 확인.

## 갤러리 · 실이미지 · 미니룸 (추가)

- **갤러리 탭**: 사진첩 모델(PhotoService 별도 인스턴스) 재사용. 업로드·대표·삭제·공개범위 동일.
- **실이미지 업로드**: 사진첩·갤러리·다이어리에 `<input type=file>` + JS `FileReader` →
  data URI hidden 필드 → 일반 POST. 실제 이미지가 `<img>`로 렌더(없으면 placeholder).
- **미니룸**: 메인화면 Mini Room 영역에 아이소메트릭 3D 픽셀아트 방을 인라인 SVG로 노출
  (바닥 격자·벽·아치창·러그·가구·미니미). `render_miniroom()`의 `_iso()`/`_box()` 헬퍼.
  - viewBox를 콘텐츠 bbox(90 -64 420 312)에 타이트하게, 높이 270px 고정으로 한 화면에 수용.
  - **표시 우선순위**: 업로드 이미지 > 고정 파일(`static/miniroom.png`) > 기본 SVG.
  - **업로드 영속화**: 업로드 시 data URI를 디코딩해 `static/miniroom_uploaded.<ext>`로 저장,
    서버 시작 시 `_find_uploaded()`로 복원 → **재접속·재시작에도 유지**. reset 시 파일 삭제.
    (페이지엔 거대 data URI 대신 `/static/...` URL만 들어가 가벼움.)
  - **벽지 꾸미기**: 6색 스와치(하늘/핑크/민트/레몬/라일락/그레이) 선택 → `/wallpaper?w=` →
    미니룸 배경·SVG 벽 색 즉시 반영(주인만). 고정 이미지 모드에서도 letterbox 배경에 적용.
  - 고정 파일은 `/static/<name>` 라우트로 서빙(디렉터리 탈출 차단).
- 런타임 검증: 갤러리 실이미지 `<img src=data:image...>` 렌더, 다이어리 `.dimg`, 미니룸 SVG
  polygon 42·창문 3·미니미 좌표 정상(NaN 없음) 확인.

## 범위/잔여 리스크

- 시점은 데모용 3종 고정(주인/대표 일촌/대표 방문자). 임의 사용자 시점은 미지원.
- 서버 상태 인메모리(재시작 시 시드로 초기화).
- 갤러리·즐겨찾기·친구찾기·BGM은 deferred(placeholder/링크만).
- 정적 데모(`render_minihome_demo.py`)는 스냅샷용으로 유지, 인터랙션은 `serve.py`가 담당.
