# 보안 계획 (Security Planning)

> 1단계 '구조화' 산출물입니다. `00_sources/02_requirements/realfield-부동산실거래.md`의
> SECR 보안 요구사항을 검증 가능한 기준으로 정리합니다.

## 보안 요구사항 (SECR)

| ID | 명칭 | 내용 | 중요도 |
| --- | --- | --- | --- |
| SECR-001 | 인증키 비노출 | `serviceKey`를 소스코드·컨테이너 이미지·설정 파일에 평문으로 두지 않음. 환경변수(`MOLIT_SERVICE_KEY`)로만 주입 | 필수 |
| SECR-002 | 설정 외부화 | 인증키·엔드포인트·회복력 정책을 Config Server로 외부화, 비밀값은 환경변수 참조 | 필수 |
| SECR-003 | 인증키 로그 마스킹 | 로그·트레이스·에러 응답에 인증키가 출력되지 않음(마스킹) | 필수 |
| SECR-004 | 개인정보 미보유 확인 | 적재 데이터에 거래 당사자 식별정보가 없음을 검증. 동·매도자/매수자 구분은 비식별 범주값 | 필수 |
| SECR-005 | 키 만료·교체 | 인증키 만료·교체 시 무중단 반영 가능(Config 설정 갱신) | 선택 |

## 인증키 관리

- data.go.kr `serviceKey`는 개인 발급 자격증명입니다. 코드·이미지·Config 파일·git에 평문 저장을 금지합니다.
- 주입 경로는 환경변수 `MOLIT_SERVICE_KEY` 하나로 통일합니다. Config 서버는 `${MOLIT_SERVICE_KEY:}`로 참조만 합니다.

## 데이터 보호

- 부동산 실거래가는 공개 데이터이며 개인 식별정보를 포함하지 않습니다(동·호 정보 비제공).
- 수집·적재 전 구간에서 추가 개인정보가 섞이지 않도록 표준 스키마(AptTransaction)만 적재합니다.

## 감사·추적

- 수집 트리거(시군구·계약월·요청자)와 적재 건수를 감사 로그로 남깁니다.
- 외부 호출 실패·서킷 개방 이벤트를 운영 지표로 노출합니다.

## 위협·통제 요약

| 위협 | 통제 | 근거 |
| --- | --- | --- |
| 인증키 유출 | 환경변수 주입, git 제외, 로테이션 절차 | SECR-001, SECR-002 |
| 외부 API 장애 전파 | 서킷브레이커 격리, 부분 수집 허용 | AC-2 |
| 중복·변조 적재 | 자연키 멱등 upsert, 표준 스키마 검증 | AC-4 |
| 개인정보 혼입 | 표준 스키마(AptTransaction)만 적재, 개인 식별 필드 없음 | SECR-004 |
