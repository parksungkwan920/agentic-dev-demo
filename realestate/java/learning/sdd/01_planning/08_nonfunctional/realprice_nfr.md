# 비기능 수용기준 (NFR · 성능·회복력·제약)

> 1단계 '구조화' 산출물입니다. `00_sources/02_requirements/realfield-부동산실거래.md`의
> PER·SIR·CONR 비기능 요구를 검증 가능한 기준으로 정리합니다.

## 성능·회복력·가용성 (PER)

| 항목 | 목표 | 검증 위치 |
| --- | --- | --- |
| 조회 성능 (PER-001) | 시세 통계 조회 P99 300ms | analytics read model 분리 (AC-5) |
| 수집 처리량 (PER-002) | 시군구당 1만 건/분, numOfRows=1000 페이징 | ingestion 배치 |
| 조회·수집 부하 격리 (PER-003) | 배치 수행 중 조회 P99 열화 없음 | CQRS 읽기 분리 (AC-5) |
| 동시성·트래픽 한도 (PER-004) | 다중 시군구 수집 시 일일 10,000건 한도 이내 | 호출 조절 + CONR-001 |
| 회복력 (SIR-005) | 외부 API 장애 격리, 부분 수집 허용 | resilience4j 재시도·서킷 (AC-2) |
| 멱등성 | 재수집 중복 0 | 자연키 유니크 제약 (AC-4) |
| 가용성 (PER-005) | 99.9%, 무중단 배포·롤백 | 디스커버리 + 게이트웨이 헬스체크 |

## 제약사항 (CONR)

> 설계와 구현이 반드시 준수해야 하는 외부 제약입니다.
> CONR-003(콤마 금액)·CONR-005(해제거래)는 AC-3 수용기준의 직접 근거입니다.

| ID | 내용 | AC 연결 |
| --- | --- | --- |
| CONR-001 | 외부 API 일일 트래픽 한도 10,000건(개발) 이내 동작 | PER-004 |
| CONR-002 | 외부 응답은 XML, 모든 값 문자열. 숫자·날짜 형변환은 시스템 책임 | AC-1 |
| **CONR-003** | **거래금액은 만원 단위, 천 단위 콤마·선행 공백 포함. 원 단위 변환은 시스템 책임** | **AC-3 (콤마 금액 변환)** |
| CONR-004 | 동·등기일자는 등기 완료 건에만 채워짐. 미완료 건 결측 허용 | AC-1 |
| **CONR-005** | **계약 해제는 원천에서 사후 표기(cdealType). 재수집 시 해제 상태 변경 반영** | **AC-3 (해제거래 제외), AC-4** |
| CONR-006 | Java 21 / Spring Boot 3.5.x / Spring Cloud 2025.0 / Gradle 멀티모듈 고정 | 전 구간 |
| CONR-007 | 회복력 라이브러리는 resilience4j로 한정 | AC-2 |

> **핵심 정합 규칙**: CONR-003과 CONR-005가 AC-3의 두 수용기준("콤마 금액 변환·해제거래 제외")을 강제한다.
> 인증키는 코드·이미지·Config 파일에 평문으로 두지 않습니다. 환경변수로만 주입합니다.
